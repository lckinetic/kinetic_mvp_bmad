## Epic Summary

Epic 1 delivered the intended foundations:

This aligns with the Epic 1 goal from planning artifacts: enforce API/services/adapters boundaries and baseline safety controls.

## What Went Well

- Boundary clarity improved quickly once a single source of truth was introduced in `docs/architecture-layers.md`.
- Guardrail tests were added early (`test_api_layer_no_adapter_imports.py`, `test_secrets_redact.py`) and stayed green.
- Code review surfaced meaningful edge cases (centralizing redaction via `step_fail`) and those were closed in-story.
- Documentation and implementation stayed aligned (`architecture.md`, `architecture-principles.md`, story files, sprint status).

## Challenges and Friction

- Initial redaction coverage was distributed; template paths could bypass intended controls until `step_fail` became the central choke point.
- Some sequencing metadata (timestamps/status transitions) needed cleanup during review synchronization.
- Current import boundary guard is syntax-based and does not detect dynamic import paths.

## Key Learnings

1. Conventions become reliable only when converted into executable checks.
2. Security/safety controls should be centralized at shared write points, not repeated at call sites.
3. Story files are effective as operational records when review outcomes and deferred items are captured immediately.
