# Epic 7 Retrospective (Docker, Local Env, and Release Discipline)

## Epic Review

### Scope reviewed

- `7-1-compose-stack-for-app-db`
- `7-2-ci-smoke-for-workflow-api`

### What went well

- Added a practical Docker Compose baseline for local reproducibility with clear Postgres health checks and optional app profile support.
- Container and non-container local paths stayed aligned by keeping `.venv` usage and compose usage compatible instead of forcing one runtime mode.
- CI now protects workflow API contract parity in MOCK_MODE with a focused smoke suite that remains fast and deterministic.
- Contributor guidance was strengthened by documenting CI parity commands in the onboarding path.

### What was challenging

- Environment consistency required careful wiring between local `.env`, compose networking, and runtime `DATABASE_URL` overrides.
- CI scope had to stay small enough for fast feedback while still covering the highest-value workflow lifecycle surface.
- Documentation drift risk remained present because runtime behavior now spans direct local runs, compose profiles, and CI commands.

### Key lessons

- Optional compose profiles are an effective bridge strategy when teams need both reproducible containers and direct local debugging.
- Focused smoke checks are more sustainable than broad suites for release-discipline gates; deterministic MOCK_MODE remains critical.
- Story artifacts plus onboarding docs should be updated together for any environment or CI change to keep onboarding friction low.
